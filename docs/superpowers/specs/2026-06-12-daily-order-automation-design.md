# 일일 오더 자동화 시스템 — 설계 문서

**작성일:** 2026-06-12  
**상태:** 승인 대기

---

## 1. 문제 정의

매일 아침 9개 시장 담당자가 각자 양식의 오더 엑셀을 공유 폴더에 업로드한다.  
담당자(본인)는 이 파일들을 취합해 시장별 합계·품목별 집계 자료를 만들고,  
검토 후 ERP에 수동 입력하여 발행한다.

**반복·수동 작업 지점**
- 파일마다 다른 양식을 눈으로 읽어 하나의 자료로 취합
- 거래처명 약어·오타 확인
- 합계·투입월 검증
- ERP 입력 (복붙 가능 영역 확인 후 수기 입력)

---

## 2. 목표

공유 폴더에 모인 담당자 엑셀들을 한 번의 실행으로 처리해 **검증된 일일 오더 파일** 을 생성하고, ERP 붙여넣기 준비까지 자동화한다.

### 포함 범위
- 담당자별 엑셀 파싱 (9개 시장 × 고정 양식)
- 거래처명 검증 (ERP 마스터 + 약어사전 기반 퍼지 매칭)
- 합계·투입월 이상 탐지
- 출력: 일일 오더 엑셀 (4개 시트)
- 실행: 더블클릭 `.bat` 또는 `.exe`

### 제외 범위 (1단계)
- ERP 화면 자동화 (RPA)
- ERP 자동 발행
- 이메일 자동 수신

## 4. 출력 파일 시트 구성

### 시트 ① 검증 결과
| 컬럼 | 내용 |
|------|------|
| 시장 | 담당자/시장명 |
| 행번호 | 원본 파일 위치 |
| 오류유형 | 거래처불일치 / 합계오차 / 투입월이상 |
| 원본값 | 입력된 값 |
| 추천값 | 마스터 기반 추천 (퍼지 매칭 결과) |
| 유사도 | % |

- 빨간(오류): 유사도 70% 미만, 반드시 확인
- 노란(경고): 유사도 70~90%, 자동 매칭했으나 확인 권장
- 초록(정상): 유사도 90% 이상

### 시트 ② 일일 오더 요약
시장별 → 거래처별 → 중량/금액 정리. 검토용 뷰.

| 시장 | 거래처명 | 투입월 | 품목 | 중량(톤) | 금액 |
|------|---------|--------|------|---------|------|

### 시트 ③ 품목별 집계
오늘 전체 품목별 총 수령량.

| 품목 | 총 중량(톤) | 시장 내역 |
|------|-----------|---------|

### 시트 ④ ERP 붙여넣기
ERP 컬럼 순서에 맞춘 데이터. **이 시트 전체 복사 → ERP Ctrl+V → 저장.**

> ⚠️ ERP 컬럼 순서는 실제 ERP 화면 확인 후 `output_builder.py`에 세팅 필요 (구현 전 1회)

## 5. 거래처 검증 로직

```
입력값
  → 정확 일치? → 통과 (초록)
  → 약어사전 존재? → 매핑 적용 (초록)
  → rapidfuzz 유사도 계산
      ≥ 90% → 자동 매칭 (초록)
      70~90% → 경고 표시, 추천값 제시 (노란)
      < 70%  → 오류 표시, 수동 확인 요구 (빨간)
```

사용자가 빨간 항목을 수동으로 확정하면 → 약어사전에 자동 추가 (다음날부터 초록 처리).

---

## 6. 실행 흐름

```
run.bat 더블클릭
  1. 공유폴더 경로에서 오늘 날짜 엑셀 파일 탐지
  2. 파일명/시트명으로 담당자 파서 자동 선택
  3. 각 파일 파싱 → 표준 데이터 구조로 변환
  4. 거래처 검증 (마스터 + 약어사전)
  5. 합계·투입월 검증
  6. 집계 (시장별, 품목별)
  7. 출력/ 에 오더_YYYY-MM-DD.xlsx 생성
  8. 파일 자동으로 열림
  9. 사용자: ①검증 확인 → ④ERP붙여넣기 복사 → ERP Ctrl+V → 저장·발행
```

## 7. 담당자별 파서 세팅 (구현 전 필요한 정보)

각 담당자 양식에서 아래 항목의 **컬럼 위치** 를 1회 확인해야 한다.

| 필요 정보 | 예시 |
|---------|------|
| 헤더 행 위치 | 3행 |
| 거래처명 컬럼 | C열 |
| 투입월 컬럼 | B열 |
| 품목 컬럼 | D열 |
| 중량 컬럼 | E열 |
| 금액 컬럼 | F열 |
| 합계 행 위치 | 마지막 행 or 특정 키워드 |

**→ 구현 시작 전, 각 담당자 샘플 엑셀 1부씩 제공 필요**
