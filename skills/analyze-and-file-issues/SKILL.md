  gh CLI를 사용해서 bug / enhancement 라벨로 구분해 이슈를 등록하며, gh CLI가 없으면 설치 안내를 먼저 제공한다.

## 목표

현재 세션에서 작업한 내용 전반을 검토해 **버그(bug)**와 **개선점(enhancement)**을 도출하고, GitHub 이슈로 자동 등록한다.

---

## 실행 단계

### 1단계: 컨텍스트 파악

현재 세션의 작업 내용을 파악한다:
- 수정하거나 생성한 파일 목록
- 스케줄드 태스크 또는 설정 파일 (SKILL.md, CLAUDE.md 등)
- 실행 중 발생한 에러 또는 경고 메시지
- 사용자가 직접 언급한 불편함이나 제약사항

### 2단계: 이슈 도출

아래 기준으로 이슈를 체계적으로 발굴한다:

**버그 (bug 라벨)**
- 실제로 재현된 에러 (명령어 실패, 파일 생성 실패 등)
- 잘못된 값 하드코딩 (연도, 경로, 사용자명 등 변경될 수 있는 값)
- 플랫폼 / 환경 불일치 (PowerShell vs Bash, OS 차이 등)
- 예외 처리 없이 실패할 수 있는 지점

**개선점 (enhancement 라벨)**
- 중복 실행 / 동시성 문제 처리 부재
- 에러 발생 시 fallback 또는 사용자 안내 없음
- 리소스 무한 누적 (로그, 파일, 캐시 정리 없음)
- 특정 지역·언어·환경 편향 (예: 영문 검색만, 특정 OS만 지원)
- 사용자 편의를 높이는 기능 (가이드, 자동화, 알림 등)

각 이슈는 다음 정보를 포함한다:
- **제목**: `[Bug]` 또는 `[Enhancement]` 접두사 + 한 줄 요약
- **본문**: 현상 / 원인 / 수정 방향 (3섹션 마크다운)

### 3단계: 사전 확인

이슈 등록 전에:
1. `gh auth status`로 gh CLI 로그인 여부 확인
   - 로그인 안 된 경우: `gh auth login` 안내 후 중단
   - gh CLI 없는 경우: `winget install GitHub.cli` 안내 후 중단
2. `git remote -v`로 GitHub 레포 URL 확인
   - 레포가 없으면 사용자에게 레포 URL 직접 입력 요청
3. 기존 이슈 목록 확인 (`gh issue list --repo <OWNER/REPO>`) — 중복 방지

### 4단계: GitHub 이슈 등록

각 이슈를 개별 파일로 저장 후 `--body-file` 플래그로 등록한다 (인코딩 문제 방지).

```powershell
# 각 이슈 본문을 임시 파일로 저장
$body = "## 현상`n...`n## 원인`n...`n## 수정 방향`n..."
$tmpFile = "C:\Users\$env:USERNAME\AppData\Local\Temp\gh_issue_$i.txt"
$body | Out-File -FilePath $tmpFile -Encoding utf8

# 이슈 등록
gh issue create `
  --repo OWNER/REPO `
  --title "[Bug] 제목" `
  --label "bug" `
  --body-file $tmpFile
```

> **주의**: `--body` 플래그에 줄바꿈·특수문자가 포함된 긴 텍스트를 직접 넣으면 파싱 오류가 남. 반드시 `--body-file`을 사용한다.

### 5단계: 결과 보고

등록 완료 후 표 형태로 요약 출력:

```
| #  | 제목                        | 종류         | 링크     |
|----|---------------------------|--------------|----------|
| 1  | Start-Process Bash 실패   | 🐛 Bug       | #1 링크  |
| 2  | 연도 하드코딩              | 🐛 Bug       | #2 링크  |
| 3  | 한국 뉴스 누락             | ✨ Enhancement | #3 링크 |
```

버그가 있으면 "지금 바로 수정할까요?" 라고 물어본다.

## 이슈 품질 기준

- 제목은 `[Bug]` 또는 `[Enhancement]`로 시작
- 본문은 **현상 / 원인 / 수정 방향** 3섹션 필수
- 모호한 이슈 (예: "코드가 복잡함") 는 등록하지 않는다
- 이미 등록된 이슈와 중복되면 건너뛴다
- 이슈는 최소 2개 이상, 최대 10개 이내로 압축
