## 목표

한 사실은 한 문서에만. 중복 없이, 최소로, 정확하게.

---

## 문서별 역할 (canonical home)

| 문서 | 담는 것 | 담지 않는 것 |
|------|---------|-------------|
| `CLAUDE.md` | Claude가 명령어에서 쓰는 경로·스킬 동작 규칙·환경 제약 | 프로젝트 소개, 철학 |
| `README.md` | 사람용 — 기능 설명·설치·사용법·레포 링크 | 운영 규칙, 철학 |
| `SOUL.md` | 왜 이렇게 만들었는가·판단 기준 | 사실 목록, 사용법 |
| `skills/*.md` | 스킬 실행 절차 (트리거·단계·규칙) | 다른 스킬의 내용 |

## 실행 단계

### 1단계: 모든 문서 읽기

다음 파일을 전부 읽는다:
- `CLAUDE.md`, `README.md`, `SOUL.md`
- `skills/` 하위 모든 `*.md` / `SKILL.md`

### 2단계: 중복 맵 작성

읽은 내용에서 **동일한 사실**이 두 개 이상의 파일에 있는 것을 표로 정리한다.

"동일한 사실"의 기준:
- 같은 경로·URL·수치가 여러 파일에 있음
- 같은 규칙이 표현만 다르게 반복됨
- 같은 기능 설명이 Claude용 파일과 사람용 파일 양쪽에 있음

출력 형식:
```
| 사실 | 현재 위치 | 남길 곳 |
|------|-----------|---------|
| 레포 URL | CLAUDE.md + README.md | README만 |
| ...  | ...       | ...     |
```

### 3단계: 스킬 파일 하드코딩 점검

스킬 파일 안의 예시 코드·명령어에서 아래를 확인한다:
- 날짜 하드코딩 (예: `2026-06-11` 같은 실제 날짜가 예시 코드 안에 있음)
- 사용자 이름 하드코딩 (`C:\Users\Admin\` 등은 경로로 OK, 그러나 `USERNAME` 플레이스홀더가 실제 이름으로 치환된 경우)
- 연도 고정 검색 쿼리

발견 시 `YYYY-MM-DD`, `$env:TEMP`, `오늘 날짜` 등 동적 표현으로 교체한다.

### 4단계: 파일 수정

2단계 중복 맵과 3단계 점검 결과를 바탕으로:
1. 중복 사실을 canonical home이 아닌 파일에서 제거
2. 하드코딩 값 교체
3. 제거 후 문맥이 끊기지 않는지 확인 (필요하면 한 줄 참조 추가: "자세한 내용은 CLAUDE.md 참고")

수정 전 변경 사항을 먼저 사용자에게 요약해서 보여주고, 진행 여부를 확인한다.

### 5단계: git commit

변경된 파일만 add하고 커밋한다:

```powershell
git add CLAUDE.md README.md SOUL.md
# 스킬 파일도 수정됐으면 함께 add
git commit -m "docs: remove duplicate facts, one fact one file

변경 내용 요약 (예시):
- 레포 URL → README만
- 스케줄 시간 → README만
- index.json 규칙 → CLAUDE.md만
- 브라우저 명령어 날짜 하드코딩 제거

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>"
```

## 품질 기준

- 수정 완료 후 각 파일 **50줄 이하**가 이상적 (스킬 파일 제외)
- 중복 맵의 모든 행이 해소됐을 때 완료
- 커밋 메시지에 제거된 중복 목록 포함

## 완료 보고 형식

```
## 최적화 완료

### 제거된 중복
| 사실 | 제거된 곳 | 남긴 곳 |
|------|-----------|---------|
| ... | ... | ... |

### 수정된 하드코딩
| 파일 | 수정 내용 |
|------|-----------|
| ... | ... |

커밋: [해시] [메시지]
```
